Qualtrics Workflow & Integrations


OVERVIEW

Qualtrics Workflow & Integrations enables users to integrate with the systems they already use to push and pull information and automatically trigger ticket alerts.


ROLE

  • Led the design from end to end;
  • Worked with UX researcher on usability research and customer validation.


SCOPE

When I was handed to work on Workflow & Integrations, it only allowed triggering automate ticket alerts whenever survey response matches the customer-defined conditions. We identified a usability issue with the existing condition editor and a better design solution is necessary. Meanwhile, our goal was to add the flexibility of being able to trigger events with other systems. Thus, coming up with a structure that can incorporate any type of integration workflow is the key.


CONDITION EDITOR

It depends on the level of sophistication. The old condition editor works well with very simple queries. But overall when it comes more than three level of queries, a lot of users get confused and have problems understanding the logic.

I worked with the engineer to come up with the nested All/Any condition editor. With the new design, the logic hierarchy is much clear. We use different colors to highlight each level which helps identify the logic structure. We also enable users to easily drag and drop, delete any conditions or group of conditions in the editor. The downside is that it takes much more space. We solved the issue by allowing users to collapse the whole condition set to save space when not viewing details. This design covered all the use cases and tested well with our clients.


WORKFLOW

With the old UI, users found it hard to setup a workflow that triggers events even after they went through the training. Some users complained that there was zero guidance on the interface for knowing what to do next. We realized it was very important to provide instructions on the interface to guide users through the process. To solve the issue, we divided the whole workflow into three steps. First, choose the event source which basically is the data that drives the tasks. Secondly, build some conditions that decide what part of data will trigger tasks. Lastly, setup the tasks that will be triggered.
